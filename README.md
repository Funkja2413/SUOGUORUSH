# 虎牢关塔防

一个无依赖静态塔防原型，基于 `data/level_001_hulaoguan.json` 实现核心玩法闭环。
地图动态背景层使用循环视频 `assets/hulaoguan_map_loop.mp4`，加载前回退到 `assets/hulaoguan_map.png`。
角色动画素材统一使用 `assets/characters/` 目录，命名规范见 `assets/characters/README.md`。
路径识别 mask 使用 `assets/path_mask.png`，仅用于逻辑判定，不会绘制到地图上。
战场 canvas 使用 2x 内部分辨率渲染，保持画面比例不变但提升角色与地图清晰度。

## 本地运行

```bash
python3 -m http.server 4173
```

打开：

```text
http://localhost:4173
```

## 分享链接

把本目录部署到任意静态托管服务即可，例如 GitHub Pages、Netlify、Vercel、Cloudflare Pages 或内部静态资源服务器。入口文件是 `index.html`。

## 已实现

- 吕布作为守方英雄，可死亡后复活；刘备作为最终进攻英雄，死亡后不复活
- 多路径地图与 11 个建造槽位
- 地图替换为上传的虎牢关大图
- 基于绿色参考图标注 4 个入口路径区域：左侧主路、上方中路、下方关外路、右侧营地路
- 每个出兵单位随机选择 1 个入口进入
- 5 种塔的槽位弹窗建造、升级、攻击范围与目标类型限制
- 已建塔点击弹出升级/拆除，拆除返还 80% 建造金币
- 已建塔 hover 显示简介、攻击属性与克制兵种
- 敌军沿路径寻路、抗性、奖励金币与漏怪扣血
- 6 个波次、出兵间隔、Boss 延迟登场
- 守方英雄放置在敌军道路上，可点选英雄后点击道路任意位置移动
- 选中守方英雄时不显示路径，仅通过隐藏 mask 判断可放置区域
- 守方英雄战场角色替换为上传设定图，并带脚底阴影适配地图透视
- 守方英雄非攻击、非移动放置状态下循环播放 4x3 idle 帧动画
- 守方英雄移动到新道路点时播放 4x3 walk 帧动画，到达后回到 idle
- 守方英雄普通攻击播放 4x3 attack 帧动画，替代旧攻击连线效果
- 守方英雄释放技能时播放对应 4x3 帧动画
- 西凉刀兵移动时按行进方向切换左右、正面、背面 4x3 walk 帧动画
- 西凉刀兵与英雄 1V1 出手时播放 4x3 attack 帧动画
- 守方英雄拥有独立血量，小兵靠近后只会触发 1V1 接战，未接战的小兵继续沿路前进
- 守方英雄血量归零后进入死亡状态，左下角显示 30s 复活倒计时，本局内可无限复活
- 地图、英雄、敌人、塔槽和交互判定整体 2x 渲染，HUD 仍保持 HTML 叠层
- 守方英雄普通攻击、技能 1 AOE 伤害、技能 2 魔法伤害
- 进攻英雄突进、全军加速
- 失败结算与按剩余城防计算 1-3 星
- 通关结算会按星级、剩余城防、完成时间计算分数，并写入本地 Ranking
- 参考塔防手游布局的全屏战场 HUD：左上资源、左下英雄技能、右下速度
- 真实音频素材系统：帝国时代式低音量中世纪地图氛围 BGM、每波开始 8s 号角 stinger，以及建塔/升级/拆塔、塔发射、英雄点击/攻击/死亡、小兵攻击/死亡等塔防反馈音效；素材来源见 `assets/audio/CREDITS.md`
